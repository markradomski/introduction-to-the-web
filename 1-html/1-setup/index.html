How to set up an HTML document.

The DOCTYPE

The DOCTYPE, found at line 1 of this document, basically signals the browser that this document is indeed an HTML document.

This is necessary because there are a couple languages that look a lot like HTML, and we want to make sure the browser interprets it as HTML rather than something else.

The <html> element

The <html> element represents the root of an HTML document, that's why it's also referred to as the root element. All other elements must be descendants of this element, no matter what.

That said, there are only 2 elements that are allowed as direct descendants: The <head> and the <body>

The <head>

The Head is the part of an HTML document that contains metadata about that document, such as author, description, and links to CSS or JavaScript files that should be applied to the HTML.

Think of it as the first couple pages of a book. It names the author, year of release, the publisher, a summary etc.

Meta tags

We established that meta tags exist to give the browser information about the document. However, they are not always directly relevant to the contents of the page.

For example: <meta charset="utf-8"> signals the browser on what character encoding to use. UTF-8 is the standard for the web, and you will use this character set 99.9999% of the times.

The <body>

The <body> represents the content of an HTML document. We learned that the <head> communicates to the browser. The <body> communicates towards your audience; the users.

This means all text, images, and other data you want to display should always be in the body.
