What is an HTML element?

An element is a component of the page. There are loads of pre-defined HTML elements, and they all have their own function. Putting all these components together will (hopefully) result in a

For example, the <p> element represents a paragraph. A <nav> element represents the menu (for navigation) of a page.

Let's have a look at some other interesting HTML elements.

<header> The header of a section. This is often used if the heading of a section is more than just a title.
<img> Displays an image!
<article> [This element] represents a self-contained composition in a document, page, application, or site, which is intended to be independently distributable or reusable - Quoted from MDN
<main> This element shows the main content of the document. There may be some elements on the page that do not have anything to do with the objective of the page.
for example: Let's say we got a blogging site. The navigation (menu) of my website has absolutely nothing to do with my 800 word article about Crash Bandicoot.
The article should be wrapped in a <main>, but the navigation should not. There is only one <main> allowed per document.

What happens if there is no dedicated element for my custom functionality?

If none of the existing tags seem to be suitable for your specific scenario, that's where we have to go for backup-strats.

The infamous <div> element is a "blank" element. There are no expectations or functionalities tied with it. It does nothing. this is good because this means we can tie our own functionalities to it without confusing the browser.

Note: <div> elements should only be used as a worst case scenario.

HTML Element Glossary

There are a lot of technical terms being thrown around, and they may seem confusing if you are not aware of the lingo. We got "tags", "elements", "components", "attributes", and the list goes on. Let's deconstruct an element, and see what different parts an HTML element really has.

Let's deconstruct the following: <button title="Click me and nothing will happen">Hello World</button>

<button> (opening) TAG
title ATTRIBUTE
Click me and nothing will happen ATTRIBUTE VALUE
Hello World TEXTNODE
</button> (closing) TAG
<button title="Click me and nothing will happen">Hello World</button> ELEMENT

Let's do one more: <img src="my-image.jpg" alt="An image of my beautiful face">

<img> (opening) TAG
src ATTRIBUTE
my-image.jpg ATTRIBUTE VALUE
alt ATTRIBUTE
An image of my beautiful face ATTRIBUTE VALUE
<img src="my-image.jpg" alt="An image of my beautiful face"> ELEMENT
Note: the <img> element does not have a closing tag because this element is not allowed to have descendants. (Just like the <meta> elements we saw before).

Semantics (!!!)

All predefined HTML elements exist to keep things semantically correct. Good semantics allow the browser to understand how your page is structured, and how the content is layed out.

If the browser understands your document, it means that the browser will be able to translate it back to the users.
